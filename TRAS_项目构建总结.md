# TRAS（测试报告分析系统）项目构建全过程总结

## 一、项目背景与目标
**项目全称**：Test Report Analysis System (TRAS)
**开发周期**：2026年3月
**技术架构**：Vue 3 + Vite + Element Plus (前端) / Node.js + Express + MongoDB (后端)
**核心目标**：针对复杂测试报告（多级表头、加密文件）提供深度分析、自动对比及可视化展示的集成化平台。

---

## 二、项目构建核心里程碑

### 1. 系统基础架构搭建
*   **前端工程**：使用 Vite 初始化 Vue 3 项目，集成 Element Plus UI 组件库，确立了以 `App.vue` 为核心的单页应用结构。
*   **后端服务**：基于 Express 搭建 RESTful API，使用 Mongoose 连接 MongoDB 数据库，实现了用户认证、文件上传、数据分析等核心模块。
*   **数据模型**：设计了 `User` (用户)、`Data` (文件元数据)、`Comparison` (对比记录)、`Analysis` (分析结果) 等核心 Schema。

### 2. 核心功能迭代
#### 2.1 用户认证与权限管理
*   **初始版本**：实现了基于邮箱/密码的注册与登录功能，使用 JWT 进行 Token 鉴权。
*   **逻辑升级**：响应用户需求，将登录方式从“邮箱”改为“用户名”，并增强了前后端的空值校验与错误提示。
*   **权限隔离**：
    *   **普通用户**：仅能查看、操作自己上传的文件及生成的对比记录。
    *   **管理员 (Admin)**：实现了数据穿透，可查看全系统所有用户的文件、对比记录及统计数据。
    *   **修复**：修正了 Token 负载未包含角色信息导致管理员权限失效的问题，并提供了 `promote_admin.js` 脚本用于提升用户权限。

#### 2.2 文件处理与解析
*   **多格式支持**：支持 `.xlsx`, `.xls`, `.csv`, `.json`, `.txt` 等多种格式文件的上传。
*   **加密文件处理**：攻克了 SheetJS 读取加密 Excel 的难题，实现了带密码文件的上传、预览及对比功能。
*   **多级表头解析**：开发了复杂的表头解析算法，支持处理合并单元格，确保数据读取的准确性。
*   **智能识别**：实现了从文件名自动提取“项目名称”和“报告名称”的逻辑。

#### 2.3 深度对比引擎
*   **差异检测**：实现了对两个 Excel 文件（指定 Sheet）的逐行对比，能够识别 **新增**、**删除**、**修改** 及 **移动** 的记录。
*   **自定义配置**：支持设置忽略列、主键列、表头行范围及空值处理策略。
*   **报表生成**：自动生成包含差异详情、统计摘要的 Excel 对比报告，并支持一键下载。

#### 2.4 数据可视化与看板
*   **仪表盘**：集成了 ECharts，展示文件数量、任务趋势、差异类型分布（新增/删除/修改）等关键指标。
*   **动态分析**：支持用户自定义 X/Y 轴及聚合方式（Sum/Avg/Count），实时生成数据分析图表。

### 3. 工程化与交付
*   **UI/UX 优化**：
    *   将系统标题统一为 **TRAS**。
    *   登录页采用了蓝紫色渐变背景、卡片式设计及图标引导，提升了视觉体验。
    *   移除了历史记录中不必要的“删除”和“重新对比”按钮，简化了操作流程。
*   **代码管理**：
    *   建立了 `.gitignore` 规范，过滤了 `node_modules`、`dist` 及测试数据。
    *   将代码托管至 GitHub 仓库：`https://github.com/wxgfight/TRAS.git`。
*   **打包发布**：
    *   实现了前端静态资源 (`dist`) 向后端 (`public`) 的自动同步。
    *   配置了 Express 静态文件托管与 SPA 路由回退。
    *   使用 `pkg` 工具将全栈项目打包为独立的单体 `.exe` 可执行程序，实现了开箱即用。
